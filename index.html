What Is Claude Code?

The Simple Explanation

Claude Code is a command-line tool built by Anthropic that lives inside your terminal and works across your entire workflow. It reads, writes, creates, and organizes files on your computer.

No upload limits. No session expiry. No copy-pasting back and forth.

💬 ChatGPT = asking someone for help
⚡ Claude Code = a capable teammate who actually does the work
Chat → Operating System
Without Claude Code
Chat window only Upload limits Sessions expire You do everything
With Claude Code
Full system access All files, any size Hours-long tasks AI teammates

Getting Started

How I Start Every Claude Code Project

  • Node.js 18 or higher

# Mac / Linux curl -fsSL https://claude.ai/install.sh | bash # Windows (PowerShell) irm https://claude.ai/install.ps1 | iex # Then navigate to your project cd ~/your-project-folder claude

1

Run /init

2

Review the generated CLAUDE.md

3

4

Switch to Auto-Accept

Once the plan looks right, hit Shift+Tab → Auto Accept and let it run

CLAUDE.md — Project Memory

What Goes In It
Tech Stack Directory Map Architecture Build / Test / Lint Purpose of Each Module Workflows Gotchas
  • Keep each file under 200 lines
  • Subfolder files append context — never overwrite parent
  • Reference files with @filename syntax
  • Commit to Git for team sharing
  • Add workflow rules and best practices
CLAUDE.md Best Practices
  • Run /init first then refine output
  • Be specific in instructions
  • Add gotchas Claude cannot infer
  • Keep memory concise — quality over quantity
Example CLAUDE.md
# Project: MyApp FastAPI REST API + React SPA + Postgres ## Key Commands npm run dev npm run test npm run lint ## Architecture /app → Next.js App Router pages /lib → shared utilities /prisma → DB schema & migrations ## Style Guide - Follow existing patterns - No inline styles in components ## Important Context Main data source: /data/analytics.db Deploy target: Vercel
05

Memory File Hierarchy

~/.claude/CLAUDE.md Global — applies to all projects on this machine Global
~/CLAUDE.md Parent — monorepo root context Monorepo
./CLAUDE.md Project — shared on Git with your team Shared
./frontend/CLAUDE.md Subfolder — scoped context for specific packages or modules Scoped
Rule

Subfolder files append context — they never overwrite parent CLAUDE.md files. Layer your knowledge strategically.

Skills — The Superpower

What Are Skills?

Skills are markdown guide files that Claude auto-invokes via natural language. They're reusable instruction packages that activate when relevant — no manual prompting needed.

PROJECT SKILL
.claude/skills/<name>/SKILL.md

Shared with the project via Git

PERSONAL SKILL
~/.claude/skills/<name>/SKILL.md

Personal, global across all projects

Skill Ideas
code-review testing-patterns commit-messages docker-deploy codebase-visualizer api-design linkedin-writer prd-generator data-cleaner security-reviewer
Example Skill File
--- name: testing-patterns description: Jest testing patterns for this project. Auto-activates for test-related tasks. allowed tools: Read, Grep, Glob --- # Testing Patterns Use describe + it + AAA pattern: - Arrange: set up test data - Act: call the function - Assert: check the result Use factory mocks for external deps. Never use real network calls in unit tests. ## Examples See /tests/examples/ for patterns.
Critical

The description field is what triggers auto-activation. Write it like a search query — it determines when Claude uses this skill.

Hooks — Deterministic Callbacks

Hook Types
PreToolUse PostToolUse Notification
  • PreToolUse — runs before Claude uses a tool (validate, block, log)
  • PostToolUse — runs after a tool completes (lint, format, notify)
  • Notification — fires on specific events like task completion
Permissions & Safety
"permissions": { "allow": [ "Read", "Bash(git:*)", "Write(*.evd)" ], "deny": [ "Read(env*)", "Bash(sudo:*)" ] }
Hook Configuration (.claude/settings.json)
"hooks": { "PreToolUse": [ { "matcher": "Bash", "hooks": [ { "type": "command", "command": "scripts/security-check.sh", "timeout": 5 } ] } ], "PostToolUse": [ { "matcher": "Write", "hooks": [ { "type": "command", "command": "npm run lint -- --fix", "timeout": 10 } ] } ] }

MCP — Connect Everything

What Is MCP?

Model Context Protocol is an open standard that connects Claude to your tools. Think of it as USB-C for AI — one interface for your entire tool stack. 200+ integrations, one protocol.

Prompting Techniques

Technique When To Use Example
Be Specific Always. Vague prompts = vague results "Clean this CSV — remove rows where column B is empty, deduplicate on email"
Give Examples When output format matters "Show 1–2 examples of what you want"
Chain Steps Complex multi-part tasks "First analyze, then summarize, then create action items"
Set Constraints Quality control "Max 500 words" or "Only use data from 2024"
Assign Roles Expertise-specific tasks "Act as a data analyst reviewing this dataset"
/clear Often Between unrelated tasks Resets context so old info doesn't pollute new tasks
Pro Pattern: Checkpoint + Iterate
→

Ask Claude to make a plan first — review before execution

→

Let it create checkpoints after major milestones

→

Rewind (Esc twice) if something goes wrong

→

Iterate with specific feedback until output is correct

Do's & Don'ts

✓ Do

  • Start every session in Plan Mode
  • Use Opus 4.6 with thinking enabled for complex tasks
  • Create a CLAUDE.md for every project
  • Build Skills for recurring tasks
  • Use multiple displays — keep parallel sessions visible
  • Share your CLAUDE.md with your team via Git
  • Experiment — there's no one correct workflow
  • Continuously evolve your setup as you learn

✕ Don't

  • Skip plan mode — you'll waste time fixing avoidable errors
  • Use it like a chat window — it works across your file system
  • Forget to use multiple displays for parallel sessions
  • Work in isolation — share setups with your team
  • Treat it as finished — continually evolve your workflow
  • Jump straight to execution without reviewing the plan
  • Forget to commit frequently — checkpoints save you
